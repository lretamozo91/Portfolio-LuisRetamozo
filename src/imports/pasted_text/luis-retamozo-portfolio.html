Diseña una landing page / portafolio web profesional para un desarrollador de software llamado Luis Retamozo. OBJETIVO DEL SITIO El portafolio debe funcionar para tres tipos de público: 1. Empresas y recruiters que buscan contratar un desarrollador. 2. Clientes freelance que necesitan sistemas web, apps o soluciones digitales. 3. Particulares o negocios que necesitan crear un sitio web profesional. El diseño debe sentirse profesional, moderno, tecnológico, confiable y amigable, evitando que parezca demasiado corporativo o frío. ESTILO VISUAL Usar una paleta basada en: - Gris oscuro / casi negro como color principal. - Rojo como color de acento. - Blanco o gris claro para textos principales. - Tonos grises intermedios para cards, bordes y fondos secundarios. El estilo debe ser similar a un portafolio moderno de desarrollador, con estética tecnológica, pero con detalles humanos y cercanos. PALETA SUGERIDA - Fondo principal: #111111 o #151515 - Fondo secundario: #1E1E1E - Cards: #242424 - Texto principal: #F5F5F5 - Texto secundario: #B5B5B5 - Rojo acento: #E63946 o #FF3B3B - Bordes suaves: #333333 TIPOGRAFÍA Usar una tipografía moderna, limpia y legible. Sugerencias: - Inter - Manrope - Poppins - Satoshi - Plus Jakarta Sans El diseño debe tener buena jerarquía visual, títulos grandes, textos claros y mucho espacio respirable. ESTRUCTURA DE LA PÁGINA 1. HERO / PORTADA Diseñar una primera sección impactante. Debe incluir: - Foto real de Luis en un lugar destacado. - Nombre: Luis Retamozo. - Título principal: “Desarrollo aplicaciones web y mobile para empresas, emprendedores y negocios.” - Subtítulo: “Full Stack Developer con experiencia en React, React Native, TypeScript, Node.js y desarrollo de productos digitales reales.” - Botones principales: “Ver proyectos” “Contactarme” “Descargar CV” - Elementos visuales tecnológicos: líneas sutiles, glow rojo, cards flotantes, pequeños íconos de tecnologías. - Animación sugerida: Entrada suave del texto desde la izquierda. Foto con efecto hover sutil. Cards flotantes con movimiento leve. 2. SECCIÓN “QUÉ PUEDO CONSTRUIR” Crear una sección con cards visuales para servicios. Cards: - Aplicaciones web y SaaS - Aplicaciones mobile - Sistemas a medida - Sitios web profesionales Cada card debe tener: - Ícono tecnológico. - Título claro. - Descripción breve. - Efecto hover con borde rojo o glow suave. Agregar una imagen caricaturizada de Luis en esta sección, en actitud explicativa o señalando las cards, para hacer el diseño más amigable. 3. SECCIÓN “PROYECTOS DESTACADOS” Diseñar cards grandes para proyectos. Proyectos principales: - AlquiTurno — SaaS de reservas para clubes de pádel. - Claro Pay — App mobile financiera. - Sistema Tribunal de Faltas — Sistema administrativo a medida. - ANSES / Drupal institucional — Desarrollo frontend institucional. Cada proyecto debe tener: - Imagen o mockup. - Título. - Breve descripción. - Tecnologías usadas. - Etiqueta del tipo de proyecto: SaaS, Mobile, Sistema a medida, Institucional. - Botón “Ver caso”. - Diseño tipo case study, no solo galería. Agregar animaciones: - Cards que aparecen con fade-in al hacer scroll. - Hover con elevación sutil. - Tags de tecnologías con animación ligera. 4. SECCIÓN “EXPERIENCIA PROFESIONAL” Diseñar una línea de tiempo o bloques verticales. Experiencias destacadas: - GlobalLogic — React Native Software Engineer. - GlobalLogic — Drupal Front-End Engineer. - Realmedia Network — Drupal Front-End Developer. - America Virtual — Front-End Drupal Developer. - Universidad Nacional de Salta — Programador Full Stack. Debe verse profesional y orientado a empresas. Agregar una imagen caricaturizada de Luis trabajando en notebook o revisando código. 5. SECCIÓN “STACK TÉCNICO” Diseñar una grilla organizada por categorías: Frontend: React, Next.js, TypeScript, JavaScript, HTML5, CSS. Mobile: React Native, Expo. Backend: Node.js, Express, NestJS, REST APIs. Bases de datos: PostgreSQL, MySQL, MongoDB, Redis, Prisma. Herramientas: Git, GitHub, GitLab, Jira, Render, CI/CD, Jest. Diseñar logos o pills tecnológicos. Agregar animaciones suaves en hover. 6. SECCIÓN “SERVICIOS” Crear una sección pensada para clientes freelance y particulares. Servicios: - Desarrollo de sitios web profesionales. - Desarrollo de sistemas a medida. - Desarrollo de MVP o SaaS. - Desarrollo mobile. - Integración con APIs y pagos. - Mantenimiento y mejoras evolutivas. El tono debe ser claro, comercial y confiable. Agregar una caricatura de Luis conversando con un cliente o mostrando una pantalla. 7. SECCIÓN “CÓMO TRABAJO” Diseñar un proceso en 4 pasos: 1. Relevamiento 2. Propuesta 3. Desarrollo 4. Entrega y acompañamiento Cada paso debe tener ícono, título y texto breve. El diseño debe transmitir orden, confianza y claridad. 8. SECCIÓN “SOBRE MÍ” Agregar una sección más humana. Texto base: “Soy Luis Retamozo, desarrollador de software de Salta, Argentina. Me especializo en crear soluciones web y mobile que combinan buena experiencia de usuario, arquitectura sólida y foco en resolver problemas reales.” Agregar: - Foto o caricatura amigable. - Detalles personales/profesionales: “+6 años de experiencia” “Full Stack Developer” “Técnico Universitario en Programación” “Experiencia en productos reales” “Desarrollo SaaS propio” 9. SECCIÓN “CONTACTO” Diseñar un cierre fuerte y simple. Texto: “¿Tenés una idea, proyecto o búsqueda laboral donde mi perfil pueda aportar?” Botones: - WhatsApp - Email - LinkedIn - GitHub Agregar un fondo con degradado oscuro y acento rojo. Agregar una caricatura final de Luis saludando o invitando a contactarlo. ANIMACIONES E INTERACCIÓN Incluir sugerencias visuales de animación: - Aparición suave de secciones al hacer scroll. - Hover en botones con cambio de color rojo. - Cards con elevación sutil. - Íconos con microinteracciones. - Foto principal con efecto de profundidad. - Caricaturas con pequeños movimientos o elementos flotantes. - Transiciones suaves entre secciones. USO DE IMÁGENES CARICATURIZADAS El diseño debe incluir imágenes caricaturizadas de Luis en diferentes secciones para hacerlo más cercano y amigable. Estilo de caricaturas: - Modernas. - Limpias. - Profesionales. - Amigables. - No infantiles. - Con estética tech. - Mantener coherencia visual entre todas. Ideas para caricaturas: - Luis programando. - Luis explicando una solución. - Luis mostrando una pantalla. - Luis trabajando con una app mobile. - Luis saludando en la sección contacto. - Luis señalando proyectos o servicios. IMPORTANTE: El portafolio no debe parecer genérico. Debe transmitir que Luis puede: - Trabajar en empresas. - Desarrollar proyectos freelance. - Crear sitios y sistemas para negocios. - Construir productos digitales completos. El resultado debe ser una propuesta visual moderna, responsive, clara, amigable y profesional, lista para implementar luego en React / Next.js.
